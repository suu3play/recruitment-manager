# recruitment-manager

採用候補者をローカルフォルダで管理するデスクトップアプリケーション。

## 概要

採用サイト（マイナビ・ツノル・Green・Wantedly等）から受け取ったPDFや情報を、ローカルフォルダに整理して一元管理できます。新卒・中途それぞれの選考フローに対応し、ステータス変更のたびにフォルダが自動で移動します。

## 機能

### 候補者管理
- 新卒・中途の区分別管理
- 新卒は卒業年度別にフォルダを整理
- ステータス変更時に自動でフォルダ移動
- 応募日・期限・次アクション日の管理

### 選考ステータス

| 新卒 | 中途 |
|------|------|
| 応募 → 書類選考 → 一次面接 → 最終面接 → 内定 / 不採用 / 辞退 | 応募 → カジュアル面談 → 書類選考 → 一次面接 → 最終面接 → 内定 / 不採用 / 辞退 |

### 書類管理
- PDF・Excel・Word ファイルをドラッグ&ドロップで添付
- ファイル名に「履歴書」「職務経歴書」「成績証明書」「卒業見込み」が含まれる場合は種別を自動判定
- 未判定ファイルは手動で種別を選択

### フォルダ構成

```
採用管理ルート/
├── 新卒/
│   └── 2026年卒/
│       ├── 01_応募/
│       │   └── 山田太郎_マイナビ/
│       │       ├── profile.json
│       │       └── 履歴書.pdf
│       ├── 02_書類選考/
│       ├── 03_一次面接/
│       ├── 04_最終面接/
│       ├── 99_内定/
│       └── 00_不採用_辞退/
└── 中途/
    ├── 01_応募/
    ├── 02_カジュアル面談/
    ├── 03_書類選考/
    ├── 04_一次面接/
    ├── 05_最終面接/
    ├── 99_内定/
    └── 00_不採用_辞退/
```

### 候補者詳細
- 選考ステージ履歴・メモ・評価（S/A/B/C/D）の記録
- ステージ別の担当者・日付管理
- PDFファイルのプレビュー

### 一覧・絞り込み
- 採用区分・ステータス・採用媒体でフィルター
- 期限・応募日でソート
- 期限切れ・間近の候補者をハイライト表示

### 担当者管理
- 設定画面で担当者を事前登録
- フォーム上から未登録の担当者を即時登録可能

### 中途採用の期限デフォルト
- 中途候補者の期限はデフォルトで1週間後に設定

## 技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | Electron + Vite |
| UI | React 18 + TypeScript |
| スタイル | Tailwind CSS |
| データ保存 | ローカルJSONファイル（profile.json） |

## セットアップ

```bash
# 依存パッケージのインストール
npm install

# 開発サーバー起動
npm run dev

# ビルド
npm run build
```

## 初回起動

1. 設定画面を開く
2. 「採用管理フォルダ」でデータを保存するルートディレクトリを指定
3. 必要に応じて担当者を登録
4. 候補者を追加して管理を開始
